## Scoring and Triage

### Intelligence Significance Scoring (from Marketing Lead Scoring)

Point-based system for triaging the firehose of 19+ feeds:

```
ENTITY SIGNIFICANCE SCORE (max 100)

INHERENT (max 50):
  +20  Military/government entity
  +15  Critical infrastructure proximity (<25km from power plant, military base, datacenter)
  +10  Sanctioned entity or watchlist match
  +10  Cross-domain correlation (appears in 2+ feeds)
  +5   Anomalous behavior pattern detected

CONTEXTUAL (max 50):
  +20  Matches active collection requirement
  +15  Located in priority geographic area
  +10  Temporal correlation with known event
  +5   Historical pattern match
  +3   User previously flagged similar entity

THRESHOLDS:
  60+  Push alert (immediate notification)
  40+  Daily brief inclusion
  <40  Archive for pattern analysis
```

### RICE for Intelligence Triage (from Sales Pipeline)

```
Intelligence RICE = (Scope x Severity x Confidence) / Processing Cost

Scope:     How many personas/consumers does this signal affect? (1-5)
Severity:  Operational/financial/safety impact if true (1-5)
Confidence: Source corroboration level (0.1-1.0)
Processing: Compute/analyst time to verify (0.5-5)

Portfolio:
  Quick Wins: High-confidence, low-cost (auto-generated alerts)
  Big Bets:   Wide-scope, high-severity, needs analyst investment
  Core Focus: The 3 most actionable items right now
```

### Severity x Likelihood Independence (from Legal Risk Assessment)

**Always score these independently, then multiply.** Never collapse into one intuitive number.

```
Example:
  Military aircraft over conflict zone:  Severity=5, Likelihood=2 (transponder spoofable) → Score=10
  Container ship in English Channel:     Severity=1, Likelihood=5 (routine, verified) → Score=5
  AIS-dark tanker near sanctions zone:   Severity=4, Likelihood=4 (multiple sources) → Score=16

Thresholds (S x L):
  1-4:   Accept, display with caveats
  5-9:   Flag for corroboration
  10-15: Display prominently, assign analyst
  16-25: Immediate escalation, human review required
```

### Materiality Thresholds (from Finance Variance Analysis)

Dual-threshold approach to prevent alert fatigue:

```
QUANTITATIVE thresholds (any one triggers review):
  - AIS gap > 4 hours
  - Speed deviation > 3 sigma from vessel baseline
  - Route deviation > 50nm from expected
  - Flight altitude anomaly > 5000ft from filed plan
  - Shodan ICS device count change > 20% in region/week

QUALITATIVE thresholds (always trigger regardless of quantitative):
  - Any activity in sanctioned waters
  - Any known entity of interest
  - Any military asset in unexpected location
  - Any correlation involving 3+ domains simultaneously
  - Any FIMI major wave targeting a country with active GDELT events
  - Any prediction market CONFLICT delta > 10pp co-located with military activity
  - Air raid alerts + military vessels in Black Sea simultaneously
```

## Anomaly Detection

### Variance Decomposition for Anomaly Explanation (from Finance)

Never present a black-box anomaly score. Decompose it:

```
"Vessel [name] scored 87/100 because:
  - AIS gap:        35 pts  (dark for 8 hours in shipping lane)
  - Route deviation: 25 pts  (50nm off expected course)
  - Speed anomaly:   15 pts  (loitering at 2kt in transit zone)
  - Entity history:  12 pts  (previous sanctions investigation)

  Breakdown: Behavioral 75pts + Historical 12pts
  Confidence: Medium (AIS-only, no satellite corroboration)"
```

### Benford's Law for Spoofing Detection (from Finance Audit)

Vessels transmitting fake AIS positions often have non-natural digit distributions.
Apply Benford's Law analysis to:
- First digits of latitude/longitude deltas between consecutive positions
- Speed values over time
- Heading change rates

Natural AIS data follows Benford's distribution. Spoofed positions generated by
algorithms often show uniform or patterned distributions. This is a novel detection
method no current maritime OSINT tool uses.

### Isolation Forest with OSINT Features (from Finance ML)

Replace financial features with OSINT equivalents:

```
Financial Feature        → OSINT Feature
log_amount              → log_speed_deviation (vs expected for route)
is_round_number         → is_transponder_gap (AIS went dark)
is_off_hours            → is_unusual_route (deviation from pattern)
account_z_score         → entity_z_score (behavior vs own baseline)
benfords_deviation      → position_density_deviation (expected vs actual report frequency)
```

Use TimeSeriesSplit for validation (never k-fold — prevents data leakage on temporal data).

### Reconciliation Categories (from Finance Bank Reconciliation)

When the same entity appears in multiple feeds with discrepancies:

```
Timing differences:  AIS and satellite saw same vessel, timestamps 15min apart (legitimate lag)
Source errors:       ADS-B reported wrong altitude due to sensor malfunction
Feed errors:         Ingestion pipeline dropped a field during normalization
Unrecorded items:    Vessel in AIS but no matching IMO registration
Duplicate entries:   Same aircraft from ADS-B and MLAT both tracked
Missing data:        Vessel on satellite but no AIS signal (dark vessel — the interesting case)
```

## Compliance and Legal

### Data Source Terms Compliance (from Legal Contract Review)

Every feed needs a compliance classification reviewed quarterly:

```
GREEN:  Terms explicitly allow aggregation, display, AI analysis, commercial use
YELLOW: Terms ambiguous on cross-referencing with other sources or commercial display
RED:    Terms prohibit aggregation, derivative works, or commercial redistribution

Watch for:
- Attribution requirements that conflict between providers
- Rate limits that constrain real-time use at scale
- "Non-commercial" clauses on feeds you plan to monetize
- Redistribution restrictions on derived intelligence products
- Data retention limits that affect historical analysis
```

### Escalation Triggers (from Legal Compliance)

Hard stops in automated processing — force human review when:

```
ALWAYS ESCALATE:
- Data suggests imminent threat to life (conflict data + aircraft/vessel converging)
- Entity matches sanctions list (OFAC, EU)
- All feeds show correlated anomalies simultaneously (potential coordinated event)
- Source goes dark in region where previously reliable (potential jamming/censorship)
- Data could locate a specific individual (privacy/safety concern)
- Intelligence contradicts a published assessment (could indicate deception or developing situation)
```

### Rationalization Prevention (from Legal Verification)

Train the AI assistant to never skip verification. Watch for:

```
"That's just a commercial flight"          → Verify. Commercial cover is a known tactic.
"I've seen this vessel pattern before"     → Verify. Patterns evolve.
"The ETA is obviously 3 hours"             → Calculate. Don't assume.
"This is just routine traffic"             → Compare to baseline. "Routine" is a judgment call.
"Breaking news — publish the alert now"    → Verify first. Speed ≠ accuracy.
```

## Visualization Philosophy

### Named Approaches (from Content/Creative Algorithmic Art)

Commit to a named visualization philosophy per context:

**"Operational Clarity"** (Military HUD theme)
- Every pixel reduces cognitive load. No decoration.
- Data density maximized. Color only for classification.
- Eye travels: threat → context → action in <3 seconds.
- Palette: 60% deep navy, 30% phosphor green, 10% alert red.

**"Humanitarian Witness"** (CrisisLens theme)
- Individual data points rendered as distinct entities.
- Never abstract human impact into heatmap gradients.
- The human cost is always legible.
- Palette: 60% warm light, 30% trust blue, 10% urgency orange.

**"Market Intelligence"** (Commercial theme)
- Numbers and time-sensitivity dominate.
- Charts, not maps, are primary. Map supports chart.
- Palette: 60% neutral slate, 30% brand accent, 10% data highlight.

Every new correlation pattern is a testable hypothesis:

```

spike in [market], smuggling activity follows within 72 hours"

CONTROL:   Baseline smuggling detection rate without this correlation
VARIANT:   Detection rate using this cross-domain signal
RESULT:    [Confirmed / Inconclusive / Rejected]
LEARNING:  [Adjust correlation engine weights accordingly]
```
